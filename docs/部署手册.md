# 部署手册

## 1. 部署目标

本项目支持两种常见部署方式：

- 直接用 Node.js 启动服务
- 使用 Docker / GHCR 镜像部署

项目已经包含：

- GitHub Actions CI：代码检查、测试、Docker build 校验
- GitHub Actions 多架构镜像发布：`linux/amd64`、`linux/arm64`

## 2. 关键环境变量

| 变量名 | 说明 | 是否建议配置 |
|---|---|---|
| `PORT` | HTTP 服务端口 | 是 |
| `ORACLE_USER` | Oracle 用户名 | 是 |
| `ORACLE_PASSWORD` | Oracle 密码 | 是 |
| `ORACLE_DSN` | Oracle 连接串 | 是 |
| `TEMPLATE_PATH` | Excel 模板文件路径 | 是 |
| `OUTPUT_DIR` | 报表输出目录 | 是 |
| `DOWNLOAD_ROOT` | 下载接口允许访问的根目录 | 建议与 `OUTPUT_DIR` 保持一致 |
| `REPORT_RULES_PATH` | 规则配置文件路径 | 是 |
| `REPORT_API_TOKEN` | API 鉴权令牌 | 生产环境强烈建议配置 |
| `REPORT_PUBLIC_BASE_URL` | 对外暴露的服务基地址 | 如果走反向代理或域名，建议配置 |

说明：

- 当设置了 `REPORT_API_TOKEN` 后，`/api/reports/generate`、`/api/reports/debug/unmatched`、`/api/reports/download/:fileId` 会要求带 token
- token 支持两种传法：
  - `Authorization: Bearer <token>`
  - `X-API-Token: <token>`
- `/api/health` 和 `/api/reports/sheets` 默认保持开放

## 3. 本机 Node.js 部署

安装依赖：

```bash
cd /Users/xiaochen/Desktop/123
npm ci
```

启动：

```bash
PORT=3000 \
ORACLE_USER=damoxing \
ORACLE_PASSWORD='Damoxing123!' \
ORACLE_DSN='127.0.0.1:51521/FREEPDB1' \
TEMPLATE_PATH='/Users/xiaochen/Desktop/123/模板.xlsx' \
OUTPUT_DIR='/Users/xiaochen/Desktop/123/generated' \
DOWNLOAD_ROOT='/Users/xiaochen/Desktop/123/generated' \
REPORT_RULES_PATH='/Users/xiaochen/Desktop/123/config/report-rules.json' \
REPORT_API_TOKEN='replace-with-a-strong-token' \
REPORT_PUBLIC_BASE_URL='https://report.example.com' \
npm start
```

## 4. Docker 部署

### 4.1 本地构建

```bash
docker build -t oracle-excel-report-service:local /Users/xiaochen/Desktop/123
```

### 4.2 本地运行

如果 Oracle 运行在宿主机，比如 Orbstack，本地常见连接串可以写成：

```text
host.docker.internal:51521/FREEPDB1
```

示例：

```bash
docker run --rm -p 3000:3000 \
  -e ORACLE_USER=damoxing \
  -e ORACLE_PASSWORD='Damoxing123!' \
  -e ORACLE_DSN='host.docker.internal:51521/FREEPDB1' \
  -e TEMPLATE_PATH='/app/模板.xlsx' \
  -e OUTPUT_DIR='/app/generated' \
  -e DOWNLOAD_ROOT='/app/generated' \
  -e REPORT_RULES_PATH='/app/config/report-rules.json' \
  -e REPORT_API_TOKEN='replace-with-a-strong-token' \
  -e REPORT_PUBLIC_BASE_URL='https://report.example.com' \
  -v /Users/xiaochen/Desktop/123/generated:/app/generated \
  oracle-excel-report-service:local
```

## 5. GitHub Actions 与 GHCR 发布

### 5.1 CI 工作流

文件：

```text
.github/workflows/ci.yml
```

作用：

- 安装依赖
- 执行 `npm test`
- 校验 Dockerfile 可以正常构建

### 5.2 多架构镜像发布

文件：

```text
.github/workflows/docker-multiarch.yml
```

默认行为：

- `push` 到 `main` / `master` 时发布镜像
- 打 `v*` tag 时发布镜像
- 平台为 `linux/amd64` 和 `linux/arm64`
- 默认推送到 `ghcr.io/<owner>/<repo>`

### 5.3 拉取镜像

```bash
docker pull ghcr.io/<owner>/<repo>:latest
```

如果是版本标签：

```bash
docker pull ghcr.io/<owner>/<repo>:v1.0.0
```

## 6. 接口调用示例

### 6.1 健康检查

```bash
curl http://127.0.0.1:3000/api/health
```

### 6.2 生成整本

```bash
curl -X POST http://127.0.0.1:3000/api/reports/generate \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer replace-with-a-strong-token' \
  -d '{"month":"2025-12"}'
```

### 6.3 下载生成文件

```bash
FILE_ID='把生成接口返回的 file.id 填到这里'
curl -L "http://127.0.0.1:3000/api/reports/download/${FILE_ID}" \
  -H 'Authorization: Bearer replace-with-a-strong-token' \
  -o report.xlsx
```

### 6.4 页面入口

服务启动后，浏览器访问：

```text
http://127.0.0.1:3000/
```

页面会自动读取 `/api/health` 和 `/api/reports/sheets`，并支持：

- 输入 Token
- 生成整本报表
- 生成单 sheet 报表
- 查询未匹配网点

### 6.5 MCP 接入

本项目已经提供本地 stdio MCP server：

```bash
REPORT_API_BASE_URL='https://report.example.com' \
REPORT_API_TOKEN='replace-with-a-strong-token' \
npm run mcp
```

详细说明见：

```text
docs/MCP接入说明.md
```

## 7. 上线前建议检查

建议至少确认下面几项：

1. `REPORT_API_TOKEN` 已配置，不要裸奔暴露生成接口
2. `DOWNLOAD_ROOT` 与 `OUTPUT_DIR` 一致，避免下载越权
3. 模板文件和规则文件路径都存在
4. Oracle 连接串从容器内可达
5. 先跑一份单月报表并人工抽查 9 个 sheet
6. 重点核对当前高频单位的中心归类是否符合业务口径
